Tutorial zum WSDL Generator

Man beginnt natürlich die Erstellung eines Web Services zuerst mit einer Klasse, aus der man den Web Service generieren will. Als Beispielklasse haben wir hier eine einfache HelloWorld-Klasse.

<?php /** * @webservice */ class HelloWorld {     /**      * @webmethod      * @return string      */     public function sayHello() {         return 'Hello World!';     } }

Diese Klasse besitzt genau eine Methode, die für den Web Service genutzt werden kann, nämlich die sayHello(). Es gibt keine Eingabeparameter, sondern nur einen Rückgabewert für diese Methode. Dies muss die zu generierende WSDL-datei dem Client mitteilen, damit er keine Eingabeparameter schickt um die Methode aufzurufen, aber einen String als Ausgabe erwartet, in diesem Fall den nämlich den String `Hello World!'. Diese Informationen werden über PHPDocTags (z.B. `@return string') von der Ext. Reflection API eingeholt und dem WSDLGenerator zur Verfügung gestellt. Dieser weiß dadurch, dass sayHello() einen String zurückliefert

In unserem Beispiel muss nun die Klasse WSDLGenerator eingebunden werden, um die Generierung der WSDL-datei zu starten. Das könnte also ungefähr so aussehen:

require_once 'class.WSDLGenerator.php';

Wenn der Zugriff auf beide Klassen so sichergestellt ist, kann man mit der Konfiguration des WSDLGenerators beginnen. Er benötigt den eigentlichen Servicenamen (`HelloWorldService'), einen Servicezugangspunkt (`http://www.example.org/webservices/HelloWorldService.php') und einen Namespace für die WSDL-datei (`http://www.example.org/webservices/HelloWorldService'). Außerdem ist es möglich, die Art der SOAP-Bindung anzupassen, aber standardmäßig arbeitet der Generator mit der empfohlenen Konfiguration document-wrapped. Man instanziiert als nächstes also den Generator:

$myWSDLGenerator = new WSDLGenerator(     'HelloWorldService',     'http://www.example.org/webservices/HelloWorldService.php',     'http://www.example.org/webservices/HelloWorldService');

und übergibt ihm dann die Klasse:

$myWSDLGenerator->setClass('HelloWorld');

wodurch die Bearbeitung angestoßen wird. Die fertige WSDL-beschreibung kann dann entweder als String

$myWSDL = $myWSDLGenerator->getString();

oder als DOMDocument

$myDOM = $myWSDLGenerator->getDOMDocument();

bezogen werden.
Man kann sie jedoch auch gleich in eine Datei speichern mit:

$filename = './HelloWorldService.wsdl'; $myWSDLGenerator->saveToFile($filename);

Um einen Server direkt zu starten gibt es speziell für Entwickler noch die Methode:

$myWSDLGenerator->saveToFileAndStartSOAPServer($filename); ?>
